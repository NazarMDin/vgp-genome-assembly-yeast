# 01 — HiFi Read Preprocessing with Cutadapt

## Objective

Before assembly can begin, raw PacBio HiFi reads must be screened for adapter sequences. Unlike short-read (Illumina) sequencing where adapters attach predictably to the ends of reads, SMRT sequencing can embed adapter sequences anywhere within a read due to the circular nature of the SMRT bell library preparation. A read containing an internal adapter is likely a chimeric read — two separate genomic fragments ligated together — and including it in assembly would introduce structural errors.

The goal of this step is therefore not to trim adapters off the ends of reads, but to **discard entire reads** that contain any adapter sequence anywhere within them.

---

## Background: PacBio HiFi Read Generation

PacBio HiFi reads are generated by sequencing a circular DNA molecule (the SMRTbell) multiple times with the same polymerase. The polymerase physically passes through the insert sequence and then through the adapter sequence at each end of the SMRTbell, repeating this loop multiple times per ZMW (zero-mode waveguide). The multiple passes are then used to generate a circular consensus sequence (CCS), which achieves >99% accuracy despite the lower per-pass accuracy of SMRT sequencing.

Because the sequencing is circular, adapter sequences appear between passes rather than just at the ends of reads. If the CCS algorithm or the downstream demultiplexing step fails to fully remove these internal adapters, the resulting HiFi read will contain chimeric sequence that can mislead the assembler.

---

## Tool: Cutadapt

**Version:** 4.4+galaxy0  
**Galaxy tool ID:** `toolshed.g2.bx.psu.edu/repos/lparsons/cutadapt/cutadapt/4.4+galaxy0`

Cutadapt is primarily known as an adapter trimming tool for short reads, but here it is repurposed to detect and discard HiFi reads that contain PacBio adapter sequences anywhere within them (the "anywhere" / `-b` mode).

---

## Input

| File | Format | Description |
|---|---|---|
| `HiFi_collection` | FASTA (collection) | Three synthetic HiFi read files grouped as a Galaxy dataset collection |

The three files in the collection are:
- `HiFi_synthetic_50x_01.fasta`
- `HiFi_synthetic_50x_02.fasta`
- `HiFi_synthetic_50x_03.fasta`

---

## Parameters

| Parameter | Value | Rationale |
|---|---|---|
| Mode | Single-end | HiFi reads are not paired |
| Adapter 1 (anywhere) | `ATCTCTCTCAACAACAACAACGGAGGAGGAGGAAAAGAGAGAGAT` | PacBio SMRTbell adapter sequence |
| Adapter 2 (anywhere) | `ATCTCTCTCTTTTCCTCCTCCTCCGTTGTTGTTGTTGAGAGAGAT` | Reverse complement of adapter 1 |
| Maximum error rate | 0.1 | Allows up to 10% mismatches when detecting adapter — accounts for sequencing errors in the adapter region |
| Minimum overlap | 35 bp | Requires at least 35 bp of the adapter to match before a read is flagged — prevents false positives from short chance matches |
| Search reverse complement | Yes | Ensures adapters are detected regardless of read orientation |
| Discard trimmed reads | Yes | Any read where an adapter is found is discarded entirely, not trimmed |

### Adapter Sequences

The two adapter sequences represent the two ends of the PacBio SMRTbell adapter:

```
Adapter 1: ATCTCTCTCAACAACAACAACGGAGGAGGAGGAAAAGAGAGAGAT
Adapter 2: ATCTCTCTCTTTTCCTCCTCCTCCGTTGTTGTTGTTGAGAGAGAT
```

These are the standard PacBio v3 SMRTbell adapter sequences. They are palindromic in nature (Adapter 2 is the reverse complement of Adapter 1), which is why both strands must be searched.

---

## Output

| File | Description |
|---|---|
| `HiFi_collection (trimmed)` | Filtered HiFi reads with adapter-containing reads removed |

The output retains the collection format, with one filtered FASTA file per input file.

---

## Expected Impact

For high-quality HiFi data generated with modern PacBio instruments and CCS software, the fraction of reads discarded at this step is typically very low (often <1%). However, skipping this step risks introducing chimeric sequences into the assembly, which can cause false joins between distant genomic regions — a type of misassembly that is difficult to detect without external validation data.

---

## Notes

- This step must be completed before k-mer counting and assembly, as chimeric reads would inflate k-mer counts and potentially confuse the assembler's overlap graph.
- The `HiFi_collection (trimmed)` output is used as input for both Meryl (k-mer profiling) and hifiasm (assembly) in all subsequent steps.
